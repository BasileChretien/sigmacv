# research

The consent-gated research-logging pipeline (this project is also a research vehicle: disambiguation-error and self-presentation studies).

- **`enabled.ts`** — the **master switch** `isResearchLoggingEnabled()` (env `RESEARCH_LOGGING_ENABLED`, **default OFF**). The whole programme is **paused until an IRB / ethics protocol exists**: while off, `logCvSave` writes nothing (checked before the consent read) and the consent prompt is hidden. To re-enable after approval, set the flag **and** bump `RESEARCH_CONSENT_VERSION` so users re-consent under the approved terms.
- **`log.ts`** — writes `ResearchEvent` rows. **It writes nothing unless the master switch is on AND `User.researchConsent` is true.** This gate is mandatory (GDPR + Japan APPI; logging only under IRB, confirmatory analyses pre-registered).
- **`diff.ts`** — computes the curation deltas that are the actual research signal (e.g. "mine/not-mine" corrections, composition choices), so logged events are minimal and structured rather than raw snapshots.
- **`export.ts`** — the **IRB-gated researcher export** (pure core). `researchExportGate()` is **OFF unless** the env triad `RESEARCH_EXPORT_ENABLED=true` + `RESEARCH_EXPORT_IRB_REF` + `RESEARCH_EXPORT_PSEUDONYM_SALT` (≥16 chars) is set. `buildResearchExport()` de-identifies: subjects become keyed-HMAC **pseudonyms** (never the user id), only **pre-registered** event types are emitted, and payloads are the already-minimized `diff.ts` signals (no names/emails/ORCIDs). `buildKeyTable()` produces the **separate re-identification key table (対照表)** — the `subject` pseudonym mapped back to `userId`/name/email/ORCID — which the **個人情報管理者 (personal-information manager)** holds under access control to honour withdrawal/erasure; it is **never** given to the analyst, who works from the de-identified dataset alone (protocol §4.2.1). The actual DB read + JSONL write is the **offline** operator script `scripts/export-research-data.ts` (`npm run research:export`), which writes the two files separately (the key table as `KEYTABLE-DO-NOT-SHARE_*`) — there is **deliberately no network route**. Keep `PRE_REGISTERED_EVENT_TYPES` in sync with the `type` strings written in `log.ts`. **Do not enable** until the protocol is approved and fields are finalised to the pre-registration.

Treat the flag and the consent check as hard preconditions — never log on a path that could run without them, and keep events data-minimized. The export inherits the same posture: gated off, pseudonymous, minimal.
